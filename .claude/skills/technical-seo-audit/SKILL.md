# Technical SEO Audit

## Focus areas

- Crawl paths, robots directives, XML sitemaps, pagination, redirect chains, and status code hygiene.
- Indexation controls including canonicals, noindex usage, parameter handling, and duplicate content patterns.
- JavaScript rendering risks, hydration gaps, lazy-loaded content, and critical content exposure in HTML.
- Schema markup integrity, semantic HTML, and template consistency.
- Performance signals that influence discoverability and UX, especially Core Web Vitals on mobile.

## Workflow

1. Map page types and templates.
2. Check crawl and indexation rules.
3. Inspect rendered output and metadata.
4. Validate structured data and internal linking.
5. Review performance bottlenecks and mobile parity.
6. Produce fix recommendations with validation steps.

## Must-check items

- Canonical targets are valid, self-consistent, and not conflicting with redirects or noindex.
- Important links and content exist in rendered HTML, not only after client-side interaction.
- XML sitemaps contain canonical, indexable URLs and reflect real site structure.
- Structured data matches visible content and required properties for target rich results.
- Faceted navigation and filtered URLs do not create index bloat.
